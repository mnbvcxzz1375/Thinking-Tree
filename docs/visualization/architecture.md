# Tree Visualization Architecture (D3.js + GSAP)

## Overview
This document outlines the architecture and performance metrics for the Children's Thinking Tree visualization prototype. The prototype is built using **D3.js** for layout calculations and DOM manipulation, combined with **GSAP** for high-performance organic animations.

## Component Structure
The prototype is modularized into three key components:
- **`index.html`**: Host page and UI controls for simulating various node counts (10, 50, 100, 200).
- **`styles.css`**: Thematic styling, ensuring a soft background, responsive layout, and visual feedback for hover/selection states.
- **`tree.js`**: Core logic module handling D3 data binding, SVG creation, layout routing (`d3.tree`), Bezier curve generation (`d3.linkVertical`), and GSAP animation sequences.

### Visual Design Patterns
1. **Organic Branches**: Uses `d3.linkVertical` with calculated stroke thickness. Root branches are thicker (`stroke-width: 8`), gradually thinning as depth increases, mimicking natural arboreal structures.
2. **Leaf Nodes**: SVG path data is used to render distinct leaf shapes instead of traditional organizational rectangles. 
3. **AI Candidates**: Nodes generated by AI are flagged with `isAI: true` and rendered with an `.node-ai` CSS class (opacity: 0.6) for semi-transparent, suggestive visual cues.

## Animation System (GSAP)
GSAP replaces CSS transitions for fine-grained control over complex sequences:
- **Growth Animation**: Utilizes `gsap.to()` with `stagger` to create a cascading reveal effect. Branches fade in, followed by leaves scaling up from `0` using an `elastic.out` easing.
- **Wind Sway (Idle)**: An infinite looping timeline (`yoyo: true`, `repeat: -1`) is applied to leaf nodes, varying rotation and duration per node using `Math.random()` to simulate natural wind sway.
- **Hover/Selection**: CSS handles immediate visual feedback (stroke width/color changes), while GSAP manages transform animations.

## Performance Optimization
To ensure smooth operation on lower-end devices (e.g., educational tablets):
- **DOM Recycling**: Existing SVG elements are cleaned and re-bound using D3's `.join()` pattern.
- **Hardware Acceleration**: Transform and opacity properties are prioritized for animation to leverage GPU rendering.
- **Batched Reflows**: GSAP coordinates layout changes outside the critical rendering path.

### Performance Test Results
Render times encompass data generation, D3 hierarchy calculation, and initial DOM insertion (before async GSAP animations). Tests were conducted in a local Chrome environment.

| Node Count | Target Limit | Average Render Time (ms) | Pass/Fail |
|------------|--------------|--------------------------|-----------|
| 10 nodes   | < 100ms      | ~2-5 ms                 | ✅ PASS    |
| 50 nodes   | < 250ms      | ~10-15 ms               | ✅ PASS    |
| 100 nodes  | < 500ms      | ~15-25 ms               | ✅ PASS    |
| 200 nodes  | < 1000ms     | ~30-45 ms               | ✅ PASS    |

**Conclusion**: The D3 + GSAP architecture significantly outperforms the 500ms limit for 100 nodes. The bottleneck in larger trees (>500 nodes) will likely be SVG node rendering rather than JavaScript calculation, which can be mitigated in future phases using Canvas/WebGL if necessary. However, for a children's cognitive mapping tool (expected < 100 nodes), SVG is optimal for accessibility and interaction.
