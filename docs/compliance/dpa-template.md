## 1. Definitions

**"Children's Personal Information"** means any personal information collected from a child under 13 (COPPA), under 16 (GDPR-K), or under 14 (PIPL), including but not limited to:
- Audio recordings of children's voices
- Transcripts of children's speech
- AI-generated summaries derived from children's speech
- Activity participation records
- Any data that could identify a child

**"Processing"** means any operation performed on Children's Personal Information, including collection, recording, storage, adaptation, alteration, retrieval, consultation, use, disclosure by transmission, dissemination, alignment, combination, restriction, erasure, or destruction.

**"Sub-processor"** means any third party engaged by the Processor to process Children's Personal Information on behalf of the Controller.

---

## 2. Scope and Purpose of Processing

### 2.1 Description of Processing

The Processor shall process Children's Personal Information solely for the following purposes:

| Purpose | Data Types | Processing Activities |
|---------|------------|----------------------|
| AI Speech Analysis | Audio recordings | Receive, process, return results |
| Educational Support | Transcripts, summaries | Generate, store temporarily |

### 2.2 Purpose Limitation

The Processor shall:
- Process Children's Personal Information only for the purposes specified in Section 2.1
- Not process Children's Personal Information for any other purpose without prior written consent
- Not use Children's Personal Information for model training, improvement, or development
- Not sell, rent, or otherwise commercially exploit Children's Personal Information

### 2.3 Data Minimization

The Processor shall:
- Process only the minimum amount of Children's Personal Information necessary
- Not retain Children's Personal Information beyond the time necessary for processing
- Implement technical measures to ensure data minimization

---

## 3. Obligations of the Processor

### 3.1 Processing Instructions

The Processor shall:
- Process Children's Personal Information only on documented instructions from the Controller
- Immediately inform the Controller if, in the Processor's opinion, an instruction infringes applicable data protection law
- Not process Children's Personal Information for its own purposes

### 3.2 Confidentiality

The Processor shall:
- Ensure that persons authorized to process Children's Personal Information have committed themselves to confidentiality
- Ensure that access to Children's Personal Information is limited to those who need it for the processing purpose
- Maintain appropriate access controls

### 3.3 Security Measures

The Processor shall implement and maintain appropriate technical and organizational measures to ensure a level of security appropriate to the risk, including:

**Technical Measures**:
- Encryption of Children's Personal Information in transit (TLS 1.3)
- Encryption of Children's Personal Information at rest (AES-256)
- Regular security testing and vulnerability assessments
- Access controls and authentication mechanisms
- Intrusion detection and prevention systems

**Organizational Measures**:
- Information security policies and procedures
- Regular staff training on data protection
- Background checks for personnel with access
- Incident response procedures

### 3.4 Sub-Processing

The Processor shall:
- Not engage any Sub-processor without prior written consent of the Controller
- Impose the same data protection obligations on any Sub-processor
- Remain fully liable for the acts and omissions of its Sub-processors
- Maintain a list of approved Sub-processors and make it available to the Controller

**Current Approved Sub-processors**:

### 3.5 Data Subject Rights

The Processor shall:
- Assist the Controller in responding to data subject requests
- Implement technical measures to facilitate data subject rights
- Inform the Controller of any data subject requests received directly

### 3.6 Breach Notification

The Processor shall:
- Notify the Controller without undue delay, and in any event within **24 hours**, after becoming aware of a personal data breach
- Provide the Controller with all information necessary to meet breach notification obligations
- Cooperate with the Controller in investigating and remediating the breach

**Notification must include**:
- Nature of the breach
- Categories and approximate number of data subjects affected
- Likely consequences of the breach
- Measures taken or proposed to address the breach

### 3.7 Deletion and Return

The Processor shall:
- Delete all Children's Personal Information after the processing is complete
- Confirm deletion to the Controller within **24 hours** of deletion
- Not retain any copies, backups, or archives beyond the agreed retention period
- Implement secure deletion methods that prevent recovery

**Deletion Timeline**:
- Audio recordings: Within 24 hours of processing completion
- Temporary processing data: Within 1 hour of processing completion
- All other data: As specified in the Controller's instructions

### 3.8 Audit Rights

The Processor shall:
- Make available to the Controller all information necessary to demonstrate compliance
- Allow for and contribute to audits, including inspections, conducted by the Controller
- Provide reasonable notice prior to any audit (minimum 30 days)
- Cooperate fully with any audit or inspection

## 6. COPPA Compliance

### 6.1 School Consent Limitations

The Processor acknowledges that:
- Children's Personal Information may be collected under COPPA's "school consent" exception
- School consent is limited to educational purposes only
- The Processor shall not use Children's Personal Information for any non-educational purpose

### 6.2 COPPA Requirements

The Processor shall:
- Not condition a child's participation on the disclosure of more information than reasonably necessary
- Not use Children's Personal Information for behavioral advertising
- Maintain the confidentiality, security, and integrity of Children's Personal Information
- Delete Children's Personal Information when it is no longer needed

---

## 7. GDPR-K Compliance

### 7.1 Child Consent Requirements

The Processor acknowledges that:
- Children under 16 (or lower age per member state) require parental consent
- The Processor shall verify that appropriate consent has been obtained
- The Processor shall not process Children's Personal Information without valid consent

### 7.2 GDPR Requirements

The Processor shall:
- Process Children's Personal Information lawfully, fairly, and transparently
- Collect Children's Personal Information for specified, explicit, and legitimate purposes
- Ensure data minimization and accuracy
- Implement appropriate security measures
- Cooperate with Data Protection Authorities

---

## 8. PIPL Compliance

### 8.1 Sensitive Personal Information

The Processor acknowledges that:
- Children's Personal Information is classified as "sensitive personal information" under PIPL
- Separate consent is required for processing sensitive personal information
- The Processor shall implement enhanced security measures

### 8.2 PIPL Requirements

The Processor shall:
- Process Children's Personal Information only with separate consent
- Implement strict access controls and encryption
- Conduct Personal Information Protection Impact Assessments
- Cooperate with the Cyberspace Administration of China (CAC)

### 8.3 Cross-Border Transfer Requirements

For transfers of Children's Personal Information outside China:
- CAC security assessment is required
- Standard contract with the overseas recipient
- Personal Information Protection Certification

## Appendix A: Technical and Organizational Measures

- Role-based access control
- Regular access reviews (quarterly)

- AES-256 for data at rest

- Regular key rotation (90 days)

- Firewall protection
- Intrusion detection/prevention
- DDoS protection

### A.4 Physical Security

- Secure data centers
- Biometric access controls
- 24/7 monitoring
- Environmental controls

### A.5 Incident Response

- Documented incident response plan
- 24-hour breach notification
- Regular incident response drills
- Post-incident reviews

### A.6 Business Continuity

- Disaster recovery plan
- Regular backups
- Geographic redundancy
- Recovery time objectives
